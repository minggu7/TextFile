파일질라를 사용하여 서버에 파일을 업로드한 후, 해당 서버에서 도메인과 서버의 IP 주소를 연결하는 작업을 해야 합니다. 이 작업은 보통 서버 호스팅 관리 패널(예: cPanel, Plesk)이나 DNS 설정을 통해 처리됩니다.

도메인 매핑 과정
도메인 등록:

도메인 등록 기관(예: GoDaddy, Namecheap, Google Domains)에서 도메인을 구매한 후, 해당 도메인의 DNS 설정을 관리할 수 있습니다.
DNS 설정:

DNS 설정에서 A 레코드를 추가하여 도메인 이름(www.example.com)이 특정 서버의 IP 주소(예: 198.221.123.34)로 매핑되도록 합니다.
예: A 레코드 www.example.com -> 198.221.123.34
웹 서버 설정 (예: Apache, Nginx):

웹 서버에서 도메인에 대한 요청을 적절한 디렉토리로 리디렉션하거나, 애플리케이션 서버(예: Spring Boot)가 제공하는 파일을 서빙하도록 설정합니다.
예시: Nginx 설정
만약 Nginx 서버를 사용하는 경우, nginx.conf 또는 사이트별 설정 파일에서 도메인 매핑을 할 수 있습니다.
